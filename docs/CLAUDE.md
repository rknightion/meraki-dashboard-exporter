<system_context>
Documentation source for the Meraki Dashboard Exporter. This is a **Zensical** site (config: `zensical.toml` at repo root ŌĆö despite `mkdocs`-style naming in the Makefile, this project does NOT use MkDocs; there is no `mkdocs.yml`). The site is not built in this repo: pushes to `main` touching `docs/**`, `zensical.toml`, or `scripts/**` trigger `.github/workflows/trigger-docs-sync.yml`, which fires a `repository_dispatch` into `rknightion/m7kni-net-site`, the repo that actually builds/publishes the site.
</system_context>

<critical_notes>
- **Four files are auto-generated by `scripts/generate_*.py` ŌĆö NEVER hand-edit them.** Regenerate with `make docgen` (or the individual `make docs-*` targets) after any change to collectors/config/metrics/endpoints code, then commit the regenerated output:
  - `docs/config.md` ŌåÉ `scripts/generate_config_docs.py`
  - `docs/metrics/metrics.md` ŌåÉ `scripts/generate_metrics_docs.py`
  - `docs/collectors/reference.md` ŌåÉ `scripts/generate_collector_docs.py`
  - `docs/reference/endpoints.md` ŌåÉ `scripts/generate_endpoints_docs.py`
- **No "DO NOT EDIT" banner is written into these files** ŌĆö the generators overwrite silently with no header comment, so this list (and cross-checking against `scripts/CLAUDE.md`) is the only way to know. If you hand-edit one, the next `make docgen` silently wipes your change.
- **`docs/changelog.md` is also generated, but by release-please, not by a `scripts/generate_*.py` script.** `release-please-config.json` sets `"changelog-path": "docs/changelog.md"` ŌĆö it's rewritten by the release-please GitHub Action on every release PR. Don't hand-edit it either.
- **Site nav is defined in `zensical.toml`'s `[project] nav` array**, not in any file under `docs/`. Adding a new page requires adding it there too, or it won't appear in navigation.
- **`.meta.yml` files** (`docs/.meta.yml`, `docs/collectors/.meta.yml`, `docs/metrics/.meta.yml`) supply per-section SEO/meta defaults (title, description, tags, social) via the Zensical `meta` plugin (`meta_file = "**/.meta.yml"` in `zensical.toml`) ŌĆö they apply to every page in that directory that doesn't override the corresponding frontmatter key.
- Most hand-written pages use YAML frontmatter (`title`, `description`, `tags`, `hide`) ŌĆö follow the existing pattern in a neighboring file rather than inventing a new frontmatter shape.
</critical_notes>

## STRUCTURE
- `zensical.toml` (repo root, not in `docs/`) - site config: nav, theme, plugins, markdown extensions
- `docs/index.md`, `getting-started.md`, `config.md` (generated), `deployment-operations.md`, `security.md`, `scaling-guide.md`, `integration-dashboards.md`, `changelog.md` (generated by release-please) - top-level hand-written pages (config.md excepted)
- `docs/api-call-audit.md` - point-in-time hand-maintained audit doc of Meraki API calls (dated entries, not regenerated by tooling)
- `docs/collectors/` - `index.md` (hand-written overview) + `reference.md` (generated) + `.meta.yml`
- `docs/metrics/` - `index.md` (hand-written landing page), `overview.md` (hand-written: categories/tiers/label conventions), `metrics.md` (generated) + `.meta.yml`
- `docs/reference/` - `endpoints.md` (generated HTTP endpoint table)
- `docs/observability/` - `otel.md`, `tracing.md` - hand-written OpenTelemetry/tracing docs
- `docs/development/` - `release-process.md` - hand-written, describes the release-please flow
- `docs/extending-collectors.md` - hand-written contributor guide for adding new collectors
- `docs/includes/abbreviations.md` - glossary snippet auto-appended to every page via `pymdownx.snippets` (`auto_append` in `zensical.toml`)
- `docs/overrides/main.html` - Zensical/Material theme template override (`custom_dir = "docs/overrides"`)
- `docs/javascripts/`, `docs/stylesheets/` - theme extra JS/CSS wired via `extra_javascript`/`extra_css` in `zensical.toml`
- `docs/assets/social-card.png` - social preview image referenced from frontmatter `image:` keys
- `docs/404.md`, `500.md`, `tags.md`, `robots.txt`, `.nojekyll` - special/static pages for the built site, hand-maintained but rarely touched
</file_map>

<paved_path>
## REGENERATING DOCS AFTER A CODE CHANGE
```bash
make docgen          # runs scripts/generate-docs.sh -> all 4 generators in order
make docs-config     # config.md only
make docs-metrics    # metrics/metrics.md only
make docs-collectors # collectors/reference.md only
make docs-endpoints  # reference/endpoints.md only
```
Run `make docgen` (or the specific target) any time you add/change a collector, a Pydantic config field, a Prometheus metric, or a FastAPI endpoint, then commit the diff in `docs/` alongside your code change ŌĆö reviewers rely on these files being in sync with the code.

## ADDING A NEW HAND-WRITTEN PAGE
1. Create the `.md` file under the appropriate `docs/` subdirectory with YAML frontmatter matching sibling pages.
2. Add it to the `nav` array in `zensical.toml` at repo root.
3. If it belongs to a section with a `.meta.yml`, no per-page meta is required unless you need to override a default.
</paved_path>
